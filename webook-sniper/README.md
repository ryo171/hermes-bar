# Webook Sniper 🎟️

بوت مستقل لصيد تذاكر webook.com (باك-إند SeatCloud). يفتح صفحة الحجز في متصفح Chrome، **يقرأ كل الأقسام والأسعار والمقاعد المتاحة تلقائياً**، وتختار أنت السعر والكمية فقط — والبوت يمسك المقاعد بآلية الحجز الأصلية للموقع.

> بديل نظيف من الصفر عن الإضافة القديمة. يشتغل على الماك الآن، وقابل للتشغيل على سيرفر لاحقاً (لهيرميس).

---

## ليش بوت مستقل (Playwright) وليس إضافة؟

SeatCloud يخفي قائمة المقاعد داخل الـ iframe (خلف الكاش و service worker)، فالإضافة ما تقدر تقرأها. لكن Playwright **يقرأ طلبات الشبكة للـ iframe كاملة** — ومنها قائمة المقاعد الحقيقية لأي فعالية. لذلك يعمّم على كل الفعاليات بدون تخمين أسماء الأقسام.

---

## التثبيت (مرة واحدة)

تحتاج **Node.js 18+** على الماك. ثم:

```bash
cd webook-sniper
npm install          # يثبّت Playwright + متصفح Chromium تلقائياً
```

إذا ما تثبّت المتصفح تلقائياً:

```bash
npx playwright install chromium
```

## التشغيل

### الطريقة السهلة (تفاعلي)

```bash
node sniper.mjs "https://webook.com/ar/SA/JED/music-events/events/EVENT-NAME/book"
```

يصير التالي:

1. يفتح Chrome. **أول مرة فقط**: سجّل دخولك في webook (يُحفظ في مجلد `.profile` فما تحتاج تسجّل مرة ثانية).
2. يقرأ القاعة ويطبع كل الأسعار والأقسام وعدد المقاعد المتاحة.
3. يسألك: **السعر** و**عدد التذاكر**.
4. يمسك المقاعد ويطبع أسماءها — وتكمل الدفع يدوياً في نافذة Chrome.

### بأوامر مباشرة (أسرع وقت النزول)

```bash
# احجز 3 تذاكر سعر 450، وكرر لين يمسك
node sniper.mjs "<event-url>" --price 450 --count 3 --loop

# بس اعرض القاعة (أسعار/أقسام/متاح) بدون حجز
node sniper.mjs "<event-url>" --dump
```

| الخيار | الوصف |
|---|---|
| `--price 450` | السعر المطلوب (اتركه للأي سعر) |
| `--count 3` | عدد التذاكر (الحد الأقصى = حد الموقع، ~5) |
| `--section C6` | تحديد قسم معيّن (اختياري، يسرّع) |
| `--loop` | يكرر المحاولة حتى ينجح |
| `--dump` | يطبع القاعة فقط بدون حجز |
| `--headless` | بدون واجهة (للسيرفر) |
| `--debug` | يطبع طلبات الشبكة (لتشخيص المشاكل) |

## كيف يشتغل (باختصار تقني)

1. **حقن محرّك** (`src/engine.js`) في الصفحة قبل تحميلها — يلتقط `chart` من SeatCloud.
2. **قراءة الشبكة**: Playwright يلتقط رد `.../event/<id>/items` = قائمة كل المقاعد وحالاتها وفئاتها. منها نبني: السعر لكل فئة، الأقسام، والمقاعد الحرة.
3. **الحجز**: يمرّر أسماء المقاعد الحرة المطابقة للسعر إلى `chart.selectObjects` — آلية الموقع نفسها (موثّقة، وتتعامل مع reCAPTCHA v3 غير المرئية بشكل طبيعي). يشذّب الزيادة ويوقف عند بلوغ العدد.
4. **احتياطي**: لو تعذّرت قراءة القائمة، المحرّك يستنبط الأقسام بالحجز التجريبي السريع.

الأسماء بصيغة `القسم-الصف-المقعد` (مثل `C6-AB-16`).

---

## ملاحظات مهمة

- **الكابتشا**: الموقع فيه reCAPTCHA v3 غير مرئية (تقييم سلوك، بدون لغز). البوت يستخدم آلية الحجز الأصلية فما يثير شبهة، لكن تجنّب سرعات جنونية بلا داعي.
- **حد الحجز**: SeatCloud يسمح بعدد محدود من الحجوزات المتزامنة (~5) — طبيعي.
- **الاختبار**: جرّبه على فعالية **فيها مقاعد متاحة** (المباعة بالكامل ما فيها شي يُحجز، طبيعي).

## للسيرفر / هيرميس (لاحقاً)

نفس السكربت يشتغل `--headless` على سيرفر. هيرميس يقدر ينده:

```bash
node sniper.mjs "<url>" --price 450 --count 3 --loop --headless
```

أو يستدعي الدوال مباشرة عبر Playwright:
`window.WebookSniper.getVenue()` / `.grabByLabels(labels, count, price)` / `.getState()`.

يحتاج السيرفر جلسة webook مسجّلة (انسخ مجلد `.profile` بعد تسجيل الدخول محلياً).
